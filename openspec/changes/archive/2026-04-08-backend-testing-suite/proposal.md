# 后端测试套件 – 提案

## 为什么

当前的后端测试主要关注功能层面的「快乐路径」，缺乏可量化的扩展性与回归保障。Maker/Checker 流程以及报表生成需要大量数据处理，但我们没有体系化方式在高并发场景下验证行为，也无法通过覆盖率追踪盲区。缺少覆盖率驱动的压测、单测与 API 测试，意味着稳定性回归有机会流入生产，Jacoco 无法出具报告，审计方也无法信任验证链路。

## 将做哪些改动

- 引入压测执行框架，模拟 Maker、Checker、报表执行等并发流程，对 Spring Boot 后端施加压力以暴露竞态与资源瓶颈。
- 构建覆盖服务、仓储与工具层的后端单测套件，接入 Jacoco 以量化覆盖率并保护关键逻辑。
- 增加自动化 API 契约测试，调用认证、报表、报表执行等全部后端接口，确保响应、侧效与错误处理符合预期。
- 将 Jacoco 报告整合进 Gradle 流程，确保每次测试都会输出覆盖率产物并在 CI 中复核阈值。

## 能力

### 新增能力

- `backend-stress-testing`：提供可配置用户池的高并发场景，覆盖登录、报表执行、提交与审批路径。
- `backend-unit-test-suite`：面向服务/仓储/工具层的分层单测，使用 Jacoco 监控覆盖率，并提供 Maker/Checker/报表数据夹具。
- `backend-api-test-harness`：面向所有后端接口的 REST 回归测试，涵盖认证、报表生命周期与审计接口的正反用例。

### 调整能力

- _无_

## 影响范围

- Gradle 构建：启用 Jacoco 插件与配置，新增压测/API 测试任务，并将覆盖率报告接入 CI 流水线。
- 后端测试源码：新增单测、压测、API 测试的包结构、夹具与工具类。
- 可能需要的测试数据或 Mock：为 H2、文件导出等外部依赖提供轻量化数据准备或桩实现。
- 文档：更新 README/wiki 的测试章节，说明如何执行各套件并解读覆盖率输出。
