# 后端单元测试规范

## 新增需求

### 需求：关键服务获得覆盖

单元测试必须覆盖认证、报表与报表执行等服务层，通过 Mock/Fake 隔离外部依赖，验证业务规则（Maker/Checker 权限、状态流转、审计日志）。

#### 场景：Maker 提交规则被校验

- **当** Maker 服务在缺少必要数据时尝试提交报表执行
- **则** 单元测试需断言服务拒绝提交并抛出正确的校验错误，而无需访问数据库。

### 需求：仓储行为得到验证

仓储层测试必须在带种子数据的 H2 上执行自定义查询、投影，确认分页、过滤、状态计算均符合预期。

#### 场景：报表查询返回预期记录

- **当** 仓储查询某 Maker 的 `READY` 状态报表
- **则** 测试断言仅返回预置的 READY 报表，且排序符合规范。

### 需求：覆盖率阈值被强制执行

Jacoco 必须对 `main` 源码集产出至少 80% 行覆盖、70% 分支覆盖；若未达标，Gradle 构建应失败。

#### 场景：覆盖率不足阻断构建

- **当** Jacoco 验证任务检测到覆盖率低于配置阈值
- **则** Gradle 构建失败，并输出包含 Jacoco 报告路径的错误信息。
